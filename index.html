Partner Batch Enricher
Orchestrator
Scales partner intelligence to the entire database. Opus orchestrator pre-fetches all data, dispatches Haiku sub-agents in batches of 5 for synthesis. Resumable after interruption.
1 Query Notion Person DB → filter Type = "Partner"
2 Resolve all HubSpot contact IDs in parallel
3 Check progress.log → skip already-enriched partners
4 Pre-fetch ALL data for batch of 5 (1 batched HubSpot call + parallel Fireflies/Slack/Web)
5 Dispatch 5 Haiku sub-agents with raw data (zero MCP tools)
6 Orchestrator writes JSON output → Notion pages
// Architecture: Opus orchestrator + Haiku workers partners = notion.query("Person DB", type="Partner") done = readLog("progress.log") queue = partners.filter(p => !done.includes(p)) for batch in queue.chunk(5) { // Pre-fetch ALL data at orchestrator level data = parallel { props = hubspot.batchGet(batch.ids, [40+ props]) // 1 call for 5 deals = batch.map(p => hubspot.assocDeals(p.id)) notes = batch.map(p => hubspot.assocNotes(p.id)) calls = batch.map(p => fireflies.search(p.email)) slack = batch.map(p => slack.search(p.name)) web = batch.map(p => webSearch(p.name)) } // Dispatch Haiku sub-agents (zero tools needed) results = batch.map(p => haiku.synthesize(data[p], template) // returns JSON ) // Orchestrator writes to Notion + logs progress results.forEach(r => { notion.updatePage(r.pageId, r.markdown) log("DONE|{email}|{meetings}|{deals}") }) }

Turns HubSpot co-sell tasks into Gmail drafts. Reads Slack for pre-written messaging, groups deals by AM, checks for active threads, adds personalization, creates HTML drafts with signature.
1 Pull NOT_STARTED co-sell tasks from HubSpot (my tasks only)
2 Fetch associated deals → filter out Channel Partnership
3 Extract Slack URL from task body → read pre-written messaging
4 Group deals by AWS AM → combine multi-deal emails
5 Check last 7 days for active AM threads → recommend threading vs. fresh
6 Present drafts for review → create Gmail drafts with signature + CC
// Pull tasks and associated deals tasks = hubspot.search("tasks", { subject: "Co-sell Initiation", owner: myId, status: "NOT_STARTED" }) deals = hubspot.assoc(taskIds, "deals") .filter(d => d.channel != "Partnership") // Read Slack messaging per task tasks.forEach(t => { slackUrl = extractUrl(t.body) t.messaging = slack.readThread(slackUrl) }) // Group by AM → combine multi-deal emails grouped = deals.groupBy(d => d.aws_am_email) // Check active conversations grouped.forEach(am => { am.activeThread = hubspot.recentEmails(am.email, 7) }) // Create Gmail drafts (after user approval) signature = gmail.fetchSignature() grouped.forEach(am => { gmail.createDraft({ to: am.email, cc: "[email protected]" + am.psmEmail, body: am.messaging + signature, contentType: "text/html" }) })

Inbox Response Suggestions
Email
Scans Gmail for unread emails from real people, reads full thread context, and creates in-thread draft replies via Chrome automation. Deduplicates against existing drafts. Never sends.
1 Search Gmail: is:unread -from:me (last 48h)
2 Check existing drafts → build skip list (no duplicates)
3 Filter: humans only (skip newsletters, no-reply, already-replied)
4 Read full thread → draft context-aware reply
5 Chrome automation: navigate to thread → Reply All → type draft → Ctrl+S
// Scan + deduplicate unread = gmail.search("is:unread -from:me", limit: 20) drafts = gmail.listDrafts() skipList = drafts.map(d => d.recipient + d.subject) // Filter for real humans needsReply = unread.filter(isHuman && !alreadyReplied && !inSkipList) needsReply.forEach(msg => { thread = gmail.readThread(msg.threadId) reply = claude.draft(thread) // tone-matched, 3-6 sentences // Chrome: create in-thread draft (not via API) chrome.navigate(`mail.google.com/#inbox/${threadId}`) chrome.click("Reply All") chrome.type(reply) chrome.key("Ctrl+S") // save as draft, never send })

// Design Patterns
What Makes This Work
⚡

Parallel Execution

Every skill launches independent data fetches in the same message. HubSpot, Fireflies, Slack, and web search run simultaneously — not sequentially.

🧠

Opus + Haiku Architecture

Opus orchestrates and handles tool calls. Haiku sub-agents do text synthesis only (zero MCP tools). Result: 100% reliability, 70% cost reduction.

🔄

Resumable Batch Processing

Progress logs track every completed record. When context windows fill up, just say "continue" — the system picks up exactly where it left off.

🔗

Custom Object Traversal

HubSpot's Partner Contact custom object (2-40826561) maps AMs to companies. Direct API calls traverse this relationship that the standard tools can't.

🛡️

Never Fabricate, Never Block

Every skill follows the same rule: if data is missing, write "—" and keep going. No hallucination, no halting. Partial intel beats no intel.

📧

Human-in-the-Loop Drafts

Email skills never send. They create drafts, show previews, and wait for approval. Automation handles the research; humans handle the send button.
